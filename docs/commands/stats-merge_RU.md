# 📊 Текстовая аналитика и слияние (`stats` и `merge`)

Эти команды построены вокруг высокооптимизированного `CountingWriter` и `bufio.Scanner`, которые точно обрабатывают символы UTF-8 Unicode, обеспечивая безошибочный подсчет слов и символов для любого языка.

## 📈 `stats`: Глубокая текстовая аналитика

Вычисляет количество строк, слов, символов и символов без пробелов в тысячах файлов параллельно, используя пул воркеров `ants`.

**Флаги:**
- `--count-lines` (По умолчанию: `true`)
- `--count-words` (По умолчанию: `true`)
- `--count-chars` (По умолчанию: `false`)
- `--count-chars-no-space` (По умолчанию: `false`)

**Пример:**
```bash
# Подсчет строк и слов во всех Go-файлах, исключая папки vendor
fflow stats -r -e .go --blacklist vendor
```

## 🔗 `merge`: Конкатенация файлов

Объединяет несколько файлов в один выходной файл. Поддерживает два различных режима:

### 1. Полный режим (По умолчанию)
Читает файлы через `CountingWriter`. Это позволяет `fflow` выводить точную статистику (строки, слова, символы) о результате слияния прямо во время записи на диск.
```bash
fflow merge -e .md -o book.md --include-path --separator "\n\n---\n\n"
```

### 2. Быстрый режим (`-f` / `--fast`)
Обходит логику подсчета и использует `io.CopyBuffer` с пулом `sync.Pool` байтовых буферов по 32 КБ. Это обеспечивает **максимально возможную пропускную способность дискового I/O**, что идеально для слияния гигабайтов логов, когда текстовая статистика не нужна.
```bash
fflow merge -e .log -o all_logs.txt -f --include-name
```

**Флаги заголовков:**
- `--include-path`: Добавляет `==== /абсолютный/путь/к/файлу.txt ====` перед каждым файлом.
- `--include-name`: Добавляет `==== имя_файла.txt ====` перед каждым файлом.
