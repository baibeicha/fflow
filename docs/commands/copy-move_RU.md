# 📦 Передача файлов (`copy` и `move`)

Движок передачи разработан с упором на безопасность, параллелизм и поддержку множественных назначений. Он использует пул горутин `ants`, размер которого равен количеству ядер вашего CPU, чтобы максимизировать скорость без перегрузки диска.

## 🛡 Разрешение коллизий
По умолчанию `fflow` **никогда не перезаписывает** существующий файл, если вы явно не передадите флаг `-w` (`--rewrite`). 
Если происходит коллизия, `fflow` автоматически генерирует безопасное уникальное имя:
1. `file.txt` ➔ `file Copy.txt`
2. `file Copy.txt` ➔ `file Copy (1).txt`
3. `file Copy (1).txt` ➔ `file Copy (2).txt`

## 🚚 `copy`: Копирование в несколько мест
Вы можете копировать файлы в несколько директорий одновременно.
```bash
fflow copy -e .pdf -d ./backup_drive_1 -d ./backup_drive_2 -d ./cloud_sync
```

## 📦 `move`: Безопасное перемещение в несколько мест
Перемещение в несколько мест потенциально опасно. `fflow` обрабатывает это безопасно:
1. Он **копирует** файл во все указанные директории.
2. Отслеживает успешность операции для каждого назначения.
3. **Только если копирование успешно во ВСЕ места**, он удаляет исходный файл.
```bash
fflow move -e .mp4 -d ./archive_1 -d ./archive_2 -w
```

## 🚩 Флаги
| Флаг | Сокр. | Описание |
| :--- | :---: | :--- |
| `--dest` | `-d` | Директория назначения (можно использовать несколько раз). |
| `--rewrite` | `-w` | Принудительная перезапись существующих файлов. |
