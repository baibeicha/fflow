# 🌍 Система локализации

`fflow` имеет полностью локализованный CLI-интерфейс, на данный момент поддерживающий **Английский (`en`)** и **Русский (`ru`)** языки. 

## 🔄 Смена языка

Используйте команду `locale` для переключения языка интерфейса. Эта настройка сохраняется между сессиями.

```bash
# Переключить на русский
fflow locale ru

# Переключить на английский
fflow locale en
```

## ⚙️ Как это работает под капотом

1. **Вшитые сообщения**: Все строки переводов хранятся в YAML-файлах и вшиваются (embed) прямо в Go-бинарник с помощью `//go:embed`. Это означает, что `fflow` не требует никаких внешних зависимостей или файлов конфигурации для работы.
2. **Персистентный конфиг**: Когда вы запускаете `fflow locale ru`, утилита сохраняет ваше предпочтение в файл `locale.yaml` в стандартной директории конфигурации пользователя вашей ОС:
    - **Linux**: `~/.config/fflow/locale.yaml`
    - **macOS**: `~/Library/Application Support/fflow/locale.yaml`
    - **Windows**: `%AppData%\fflow\locale.yaml`
3. **Автоопределение**: При первом запуске, если файла конфига нет, `fflow` по умолчанию использует `en`.

## 📝 Локализация PDF-пресетов
Система локализации также распространяется на команду `pdf`. Встроенные пресеты (такие как `AcademicReport` или `ScientificArticle`) будут автоматически использовать правильную терминологию для подписей (например, "Table - " против "Таблица - ") на основе флага `--lang`, переданного в генератор PDF, независимо от языка самого CLI-интерфейса.
