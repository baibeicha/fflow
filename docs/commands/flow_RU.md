# 🔄 Руководство по автоматизации Flow

Команда `flow` — это сердце возможностей автоматизации `fflow`. Она позволяет создавать сложные пайплайны данных, где вывод одной операции напрямую подается на вход следующей.

## 🏗 Архитектура
1. **Инициализация контекста**: `fflow` собирает исходный список файлов на основе ваших глобальных флагов (`-e`, `-r` и т.д.).
2. **Интерполяция переменных**: Разрешаются переменные окружения и кастомные шаги `var`.
3. **Выполнение шагов**: Каждый шаг обрабатывает текущий массив `[]FileInfo` и возвращает мутированный массив для следующего шага.
4. **Отслеживание прогресса**: Динамический прогресс-бар обновляется для каждого шага пайплайна.

## 📝 Доступные действия

| Действие | Описание | Обязательные аргументы | Опциональные аргументы |
| :--- | :--- | :--- | :--- |
| `copy` | Копирует файлы в назначение(я). | `dest` | `rewrite` (true/false) |
| `move` | Перемещает файлы в назначение(я). | `dest` | `rewrite` |
| `delete` | Удаляет файлы с диска. | *Нет* | *Нет* |
| `rename` | Переименовывает файлы. | *Нет* | `search`, `replace`, `prefix`, `suffix` |
| `ext` | Меняет расширения файлов. | `to` | *Нет* |
| `zip` | Создает zip-архив. | `output` | *Нет* |
| `cmd` | Выполняет shell-команду для файла. | `exec` | *Нет* |
| `var` | Определяет переменную пайплайна. | `name`, `value` | *Нет* |

## 🧪 Примеры

### Пример 1: Пайплайн "Бэкап и Санитизация" (Инлайн)
Найти все `.env` файлы, скопировать их в безопасную папку, переименовать, чтобы скрыть исходное имя, и заархивировать.
```bash
fflow flow -e .env -r \
  -c "var SECURE=./secure_envs ; copy -d ${SECURE} ; rename --prefix sanitized_ ; ext --to .txt ; zip -o env_backup.zip"
```

### Пример 2: Пайплайн обработки изображений (YAML)
Создайте файл `images.yaml`:
```yaml
env:
  OUT_DIR: "./processed_images"

steps:
  - action: var
    args:
      name: "PREFIX"
      value: "vacation_"
      
  - action: copy
    args:
      dest: "${OUT_DIR}"
      
  - action: rename
    args:
      prefix: "${PREFIX}"
      
  - action: cmd
    args:
      # Используйте {} как плейсхолдер для пути к файлу
      exec: "exiftool -all= {}" 
```
Запуск:
```bash
fflow flow -e .jpg,.png -f images.yaml
```

## ⚠️ Важные замечания
- **Деструктивные действия**: Действия `delete` и `move` необратимо изменяют вашу файловую систему. Всегда тестируйте пайплайны сначала с `copy`.
- **Синтаксис переменных**: Используйте `${VAR_NAME}` для ссылки на переменные в YAML или инлайн-строках.
- **Системное окружение**: Передайте флаг `--env`, чтобы инжектить переменные окружения вашей ОС (напр., `$HOME`, `$USER`) в контекст пайплайна.
```
