# 🌐 Глобальные флаги, фильтрация и сортировка

Почти каждая команда в `fflow` (`info`, `stats`, `merge`, `copy`, `move`, `delete`, `rename`, `ext`, `zip`, `cmd`, `pdf`, `flow`) использует мощный движок поиска файлов: `FolderSearchConfig`. 

Этот движок использует библиотеку `fastwalk` для молниеносного параллельного обхода деревьев каталогов и парсинга строк без аллокаций памяти.

## 🎯 Таргетинг файлов

| Флаг | Сокр. | Описание | Глубокое погружение |
| :--- | :---: | :--- | :--- |
| `--path` | `-p` | Целевые директории. | Принимает несколько путей. Можно использовать `-p ./src -p ./docs` или через запятую `-p ./src,./docs`. |
| `--recursive` | `-r` | Обход поддиректорий. | Если не указан, `fflow` обрабатывает только прямых потомков целевого пути. |
| `--extensions` | `-e` | Фильтр по расширению. | Например, `-e .go,.md`. Внутри использует ASCII-оптимизированный парсер `fastExt()` без аллокаций. |
| `--blacklist` | | Исключение файлов/папок. | Если директория совпадает с блэклистом, `fastwalk` немедленно пропускает её (`filepath.SkipDir`), экономя огромный объем I/O. |

## 📏 Фильтрация по размеру

Вы можете фильтровать файлы по точному размеру в байтах, используя человекочитаемые единицы. `fflow` поддерживает: `b`, `kb`, `mb`, `gb`, `tb`, `pb`.

| Флаг | Описание | Пример |
| :--- | :--- | :--- |
| `--min-size` | Игнорировать файлы меньше этого. | `--min-size 500kb` |
| `--max-size` | Игнорировать файлы больше этого. | `--max-size 2gb` |

*Примечание: Фильтры размера применяются к отдельным файлам. При использовании команды `info` для расчета размеров папок, директории, содержащие только отфильтрованные файлы, корректно покажут размер 0.*

## 🔄 Продвинутая мультисортировка

Флаг `--sort-by` позволяет объединять несколько критериев сортировки в цепочку. Синтаксис: `поле:порядок`.

**Поля:**
- `name` (По алфавиту, без учета регистра, с поддержкой Unicode)
- `size` (В байтах)
- `modtime` (Время последнего изменения)

**Порядок:**
- `asc` (По возрастанию - По умолчанию)
- `desc` (По убыванию)

**Примеры:**
```bash
# Сортировка по размеру (убыв.), затем по имени (возр.) для файлов одного размера
fflow stats --sort-by size:desc,name:asc

# Сортировка от самых новых к старым
fflow info --sort-by modtime:desc
```
*Под капотом: `fflow` использует кастомный `MultiSorter` с кэшем `sync.Map` для приведения Unicode к нижнему регистру, что гарантирует экстремально быструю сортировку миллионов файлов.*
