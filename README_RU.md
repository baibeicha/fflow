## ✨ Ключевые возможности

- **🚀 Молниеносный движок**: Использует пулы параллельных воркеров (`ants`) и обход каталогов с нулевым выделением памяти (`fastwalk`) для обработки миллионов файлов за секунды.
- **📄 Продвинутый генератор PDF**: Конвертируйте исходный код, Markdown, CSV-таблицы и изображения в красиво оформленные PDF-документы. Включает 9 локализованных пресетов (Академический, Научный, Шпаргалка и др.).
- **🔄 Автоматизация пайплайнов (`flow`)**: Выстраивайте цепочки файловых операций (копирование, переименование, архивация, выполнение команд) с помощью простых YAML-конфигов или инлайн-строк в CLI с поддержкой интерполяции переменных.
- **📊 Глубокая аналитика**: Точный расчет размеров директорий (включая вложенные), подсчет строк, слов и символов.
- **🛠 Массовые операции**: Копирование, перемещение, удаление, переименование, смена расширений и создание архивов с хирургической точностью.
- **🌍 Встроенная локализация**: Бесшовное переключение интерфейса между **английским** и **русским** языками.
- **⚙️ Гибкая настройка**: Конфигурация на базе Viper, поддержка переменных окружения и обширные CLI-флаги для любого сценария использования.

## 🚀 Установка

### Из исходного кода
Требуется **Go 1.21** или выше.

```bash
git clone https://github.com/yourusername/fflow.git
cd fflow
go build -o fflow ./cmd/fflow
# Переместите в PATH (Linux/macOS)
sudo mv fflow /usr/local/bin/ 
```

### Через Go Install
```bash
go install github.com/yourusername/fflow/cmd/fflow@latest
```

## ⚡ Быстрый старт

**1. Анализ директории и сортировка по размеру:**
```bash
fflow info ./my_project --sort-by size:desc
```

**2. Слияние всех `.log` файлов в один с заголовками:**
```bash
fflow merge -e .log -o combined_logs.txt --include-path --separator "\n---\n"
```

**3. Генерация PDF из вашего Go-кода:**
```bash
fflow pdf -e .go --preset SourceCode --lang ru -o codebase.pdf
```

**4. Бэкап и переименование файлов в одном пайплайне:**
```bash
fflow flow -c "var BACKUP=./backup ; copy -d ${BACKUP} ; rename --prefix backup_"
```

## 🌐 Глобальные фильтры и флаги

Почти все команды `fflow` поддерживают мощный набор глобальных фильтров для точного выбора нужных файлов.

| Флаг | Сокр. | Описание | Пример |
| :--- | :---: | :--- | :--- |
| `--path` | `-p` | Целевые директории (через запятую или несколько флагов). | `-p ./src,./docs` |
| `--recursive` | `-r` | Рекурсивный обход поддиректорий. | `-r` |
| `--extensions` | `-e` | Фильтр по расширениям файлов. | `-e .go,.md,.txt` |
| `--blacklist` | | Исключить конкретные файлы или папки. | `--blacklist node_modules,.git` |
| `--min-size` | | Минимальный размер файла (`b, kb, mb, gb, tb, pb`). | `--min-size 1mb` |
| `--max-size` | | Максимальный размер файла. | `--max-size 500kb` |
| `--sort-by` | | Сортировка. Формат: `поле:порядок`. | `--sort-by size:desc,name:asc` |
| `--quiet` | `-q` | Скрыть UI, прогресс-бары и цвета. | `-q` |
| `--verbose` | `-v` | Включить подробное логирование. | `-v` |

## 📄 Глубокое погружение: Генерация PDF

`fflow pdf` — это не просто конвертер текста в PDF. Это полноценный генератор документов, который понимает типы файлов (Код, Markdown, CSV, Изображения) и применяет профессиональные типографские правила.

### 🎨 Встроенные пресеты
Используйте флаг `--preset`, чтобы мгновенно применить сложные конфигурации. Каждый пресет локализован для **EN** и **RU**.

| Название пресета | Для чего лучше всего | Ключевые особенности |
| :--- | :--- | :--- |
| `A4Portrait` | Обычные списки, тексты | Стандартные поля, центрированные подписи. |
| `A4Landscape` | Широкие CSV-таблицы | Альбомная ориентация, выравнивание по левому краю. |
| `AcademicReport` | Лабораторные, курсовые | 14pt Arial, поля 20мм, подписи "Таблица - X" / "Рисунок - X". |
| `ScientificArticle` | Публикации (ВАК, РИНЦ) | Строгий формат, подписи таблиц справа, 12pt. |
| `SourceCode` | Бэкенд-код, конфиги | 9pt Courier, серый фон, отключены подписи картинок/таблиц. |
| `ServerLogs` | Большие объемы логов | Плотная альбомная верстка, 7pt, минимальные поля. |
| `Cheatsheet` | Шпаргалки, справочники | Максимальная плотность данных, 6pt, включен Markdown. |
| `A3DataHeavy` | Огромные датасеты | A3 Альбомная, префиксы "[ВЫГРУЗКА]", растягивание таблиц. |
| `PhotoAlbum` | Галереи изображений | Альбомная, подписи снизу по центру, крупный рендер картинок. |

### ⚙️ Тонкая настройка
Переопределите любой параметр пресета через CLI-флаги:
```bash
fflow pdf -e .md \
  --preset AcademicReport \
  --lang ru \
  --orientation landscape \
  --margin 15 \
  --font-size 12 \
  --code-bg-color "240,240,240" \
  --table-stretch-width \
  -o my_report.pdf
```
*(См. [docs/commands/pdf_RU.md](docs/commands/pdf_RU.md) для полного списка из 40+ флагов настройки PDF).*

## 🔄 Глубокое погружение: Пайплайны (Flow)

Команда `flow` позволяет выстраивать цепочки операций. Результат одного шага становится входными данными для следующего.

### 1. Инлайн-пайплайны в CLI
Используйте точку с запятой `;` для разделения шагов. Используйте `var` для определения переменных.
```bash
fflow flow -e .jpg \
  -c "var DEST=./backup/images ; copy -d ${DEST} ; rename --prefix vacation_ ; zip -o vacation_archive.zip"
```

### 2. YAML-пайплайны
Для сложной логики используйте YAML-файл (`pipeline.yaml`):
```yaml
env:
  BACKUP_DIR: "./daily_backups"
  PREFIX: "prod_"

steps:
  - action: copy
    args:
      dest: "${BACKUP_DIR}"
      rewrite: "true"
      
  - action: rename
    args:
      prefix: "${PREFIX}"
      
  - action: ext
    args:
      to: ".bak"
      
  - action: zip
    args:
      output: "${BACKUP_DIR}/archive.zip"
```
Запуск:
```bash
fflow flow -f pipeline.yaml --env
```
*(Флаг `--env` инжектит системные переменные окружения вашей ОС в контекст пайплайна).*

## 🌍 Локализация

`fflow` поддерживает полный перевод интерфейса. По умолчанию он определяет локаль системы или использует английский.

**Переключить на русский:**
```bash
fflow locale ru
```
**Переключить на английский:**
```bash
fflow locale en
```
*Примечание: Настройка языка сохраняется в пользовательской директории конфигурации ОС (например, `~/.config/fflow/locale.yaml`).*
